## Bug Fix: Ollama 本地向量模型连接报错
- **问题**：配置 Ollama 作为 Embedding 模型时，测试连接报错 `TypeError: Cannot read properties of undefined (reading 'sort')`
- **根因**：`electron/embedding.ts:19` 构造的 URL 为 `baseUrl + '/embeddings'`，缺少 `/v1/` 前缀。Chat 接口 (`openai-provider.ts:12`) 正确使用了 `/v1/chat/completions`，但 Embedding 接口漏掉了 `/v1/`
- **影响**：Ollama 的 OpenAI 兼容 API 在 `/v1/embeddings`，请求 `/embeddings` 返回非标准响应导致 `data.data` 为 `undefined`
- **修复**：
  1. 将 URL 改为 `baseUrl + '/v1/embeddings'`
  2. 添加防御性检查：`data.data` 为 undefined 时抛出具描述性的错误信息

## Bug Fix: 导入小说 AI 推演 JSON 解析失败
- **问题**：导入小说（10章）步骤"AI 推演全局配置与架构"报错 `AI 返回的数据格式乱码，无法解析为有效层级结构`，响应末端显示 `yItems": "（待确认）"...} } ] }`
- **根因**：全局推演的 AI 响应（novelConfig + architectureFiles + 3-8 角色卡 × 15+ 字段）远超模型默认 max_tokens（4096），响应被截断导致 `"keyItems"` 断成 `yItems"`
- **修复（4 处）**：
  1. `base-command.ts`: `callLLM`/`callLLMWithBuilder` options 增加 `maxTokens?: number`
  2. `llm-store.ts`: `generateStream` options 增加 `maxTokens`，传递到 IPC
  3. `import-novel.command.ts`: 全局推演设置 `maxTokens: 16000`；向量结果截断至 500 字符/条；章节采样从 3000 降至 2000 字
  4. `base-command.ts parseJSON`: 增加 `repairTruncatedJSON()` 截断恢复逻辑 + `looksTruncatedJSON()` 诊断 + 友好错误提示（建议增加 max_tokens）
